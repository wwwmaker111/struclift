## 效果简要（按项目）

1. **zlib**：体量小、指标最稳；O0 置信与覆盖最高；O1–O3 平均置信下降但结构指标与 O0 接近，**含 M_gt≈99%**，对齐链完整。  
2. **sqlite**：覆盖与 O0 置信都很好；O1–O2 置信偏低、O3 又回升，属 **优化级对 W 分布影响** 的常见现象。  
3. **busybox**：样本多、**含 M_gt≈98–99%**；O0 置信略低于 sqlite，整体仍健康。  
4. **openssl**：样本最多；**含 M_gt≈94–96%**（大图谱路径/缺表略多）；O1 置信相对 O2/O3 偏低，解读时宜 **分档比较**。  
5. **coreutils**：**含 M_gt≈96%**；平均置信与「高置信」比例相对适中，**低置信比例略高**，可作困难样本。  
6. **libxml2**（当时）：**未完成入表**；需修复 ELF/IR 后重新 regen 再填入上表。  
7. **curl**：每档 **1691** 样本；O0 **BB 覆盖 ~84%**、**平均置信 ~0.64**（工程大、回调多，略低于 zlib/sqlite 属预期）；已用 **`make CFLAGS=…` 保留 DWARF** 后 regen。详情见 `BINSKEL_ALIGNMENT_TABLE.md` **curl** 四行。

## 如何刷新本表（本机）

```bash
cd /mnt/e/structlift
source ~/struclift_wsl/.venv/bin/activate  # 可选
python3 scripts/report_alignment_all_datasets.py ~/structlift_datasets
python3 scripts/report_alignment_all_datasets.py ~/structlift_datasets --csv > ~/alignment_report.csv
```

将终端输出中的 **libxml2 / curl** 新数字更新进本表即可。
